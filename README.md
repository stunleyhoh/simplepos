# 简单草本 POS 系统

这是一款适合新手继续扩展的网页 POS。它可以直接在本机离线使用，也可以上传到静态网站托管平台上线。上线后浏览器会缓存核心文件，断网时仍然可以打开和收银。

## 当前演示数据

系统默认使用你的业务场景做演示：

- 简单草本减脂计划第一阶段
- 售价：RM 150
- 周期：3 个星期
- 示例条码 / SKU：`SLIM-P1-3W`

打开系统后，点击“简单草本减脂计划第一阶段”加入购物车，实收填写 `150`，再点“完成收款”，就可以看到完整的小票和今日销售记录。

下一步演示时可以填写客户资料：

- 客户姓名
- 电话 / WhatsApp

完成收款后，小票会自动显示 3 个星期服务周期，也会把客户资料一起保存到销售记录里。导出 CSV 时，这些资料也会一起导出，方便后续跟进。

## 已有功能

- 商品列表、分类筛选、名称或条码搜索
- 点击商品加入购物车
- 修改数量、清空购物车、折扣、实收、找零
- 完成收款后自动扣库存
- 小票预览和打印
- 客户姓名、电话和 3 星期计划周期记录
- 当前分行选择，销售记录自动带分行
- 管理员后台全局总览，可按分行查看销售表现
- 商品新增和同条码覆盖更新
- 今日销售汇总
- 销售记录导出 CSV
- PWA 离线缓存和安装到电脑

## 未来分行预设

当前版本已经预设了分行结构：

- 总店
- 分行 1
- 分行 2

每一笔销售都会保存所属分行。管理员进入后台后，可以总览全部分行的销售额、订单数、客户数和库存概况。

注意：当前仍然是离线单机数据，所以不同电脑或不同分行之间不会自动同步。真正的多分行使用，需要下一阶段加入云端数据库和登录系统，让各分行上传订单，总管理员再从后台查看全局实时数据。

## 本地运行

如果电脑装了 Python，在这个文件夹运行：

```powershell
python -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

也可以直接双击 `index.html` 预览，但离线安装功能需要通过 `http://localhost` 或 HTTPS 网站访问。

## 上线方式

最简单的方式是把这些文件上传到任意静态网站服务，例如 Netlify、Vercel、Cloudflare Pages、GitHub Pages，或你的服务器静态目录。

上线时至少需要上传：

- `index.html`
- `styles.css`
- `app.js`
- `sw.js`
- `manifest.webmanifest`
- `icon.svg`

注意：正式上线必须使用 HTTPS，浏览器才会启用 PWA 离线缓存。

## 数据保存在哪里

当前版本的数据保存在使用者这台设备的浏览器 `localStorage` 中，所以它天生支持离线，但不同电脑之间不会自动同步。

后续如果要做多门店、多设备同步，可以继续增加：

- 登录账号
- 云端数据库
- 库存同步
- 订单同步
- 权限管理
- 会员系统
- 扫码枪和小票打印机适配
