# 简单 POS 系统测试计划

## 1. 基础打开

- 使用 `http://localhost` 或 Firebase Hosting 地址打开系统。
- 顶部版本应显示最新版本。
- 断网后刷新，页面仍可打开。

## 2. 收银流程

- 使用已授权邮箱登录 POS。
- 选择当前分行。
- 点击 `简单草本减脂计划第一阶段` 加入购物车。
- 实收金额应自动预填为应收金额。
- 输入折扣后，应收和找零应重新计算。
- 完成收款后，应出现小票。
- 今日销售应出现订单，可用客户、电话或订单号搜索。

## 3. 后台管理员

- 使用本机配置的管理员邮箱进入后台。
- 日期范围切换为今天、本月、全部，确认全局指标变化。
- 分行表现、热销商品、每日销售趋势应跟随日期范围变化。
- 客户跟进可标记 `已联系` 和 `已完成`。
- 已完成客户不应继续显示在待跟进列表。
- 导出客户名单，确认包含计划结束日期、到期状态、跟进状态和累计消费。

## 4. 分行和授权

- 新增一个测试分行。
- 授权一个测试 POS 用户到该分行。
- 切换到新分行后，库存应独立显示。

## 5. 库存和报表

- 调整商品库存，库存流水应新增记录。
- 修改业务设置里的低库存提醒阈值，低库存列表应跟着变化。
- 导出销售记录、分行汇总、商品排行、库存总表、客户名单、审计日志、库存流水。
- CSV 用 Excel 打开时，中文应正常显示。

## 6. 离线同步

- 登录后断网。
- 完成一笔收款，顶部应显示待同步。
- 修改客户跟进状态，顶部应显示订单更新待同步。
- 恢复网络后点击补传离线订单，待同步数量应下降。

## 7. 数据保护

- 导出完整 JSON 备份。
- 从备份恢复前应出现确认。
- 清空全部数据时，必须输入 `RESET` 才会继续。
