# 简单草本 POS 系统

当前应用版本：`v0.53`

这是一款适合新手继续扩展的网页 POS。它的使用策略是“联网优先、断网兜底”：平时上线联网使用，浏览器会优先读取服务器最新版；遇到断网时，系统会自动使用本机缓存继续打开和收银。

## 当前演示数据

系统默认使用你的业务场景做演示：

- 简单草本减脂计划第一阶段
- 售价：RM 150
- 周期：3 个星期
- 示例条码 / SKU：`SLIM-P1-3W`

打开系统后，点击“简单草本减脂计划第一阶段”加入购物车，实收填写 `150`，再点“完成收款”，就可以看到完整的小票和今日销售记录。

下一步演示时可以填写客户资料：

- 客户姓名
- 电话 / WhatsApp

完成收款后，小票会自动显示 3 个星期服务周期，也会把客户资料一起保存到销售记录里。导出 CSV 时，这些资料也会一起导出，方便后续跟进。

## 未来分行预设

当前版本已经预设了分行结构，也允许管理员继续开设新分行：

- 总店
- 分行 1
- 分行 2

每一笔销售都会保存所属分行和收银员。管理员进入后台后，可以总览全部分行的销售额、订单数、客户数和库存概况。

管理员可以在后台完成这些动作：

- 新增分行
- 为每个分行委任授权 POS 使用者
- 查看每位使用者所属分行
- 导出销售记录，记录中会包含分行和收银员

注意：当前已经按“未来多分行”预设数据结构，但订单仍保存在本机浏览器里。真正的联网多分行使用，需要下一阶段加入云端数据库和登录系统，让各分行在线上传订单；断网时先暂存在本机，恢复网络后再同步到总后台。

## POS 使用者

收银前，需要使用管理员授权的账号登录 POS。授权用户只能在所属分行完成收款。

登录规则：

- 联网时：使用 Google 登录验证身份。
- 断网时：使用授权邮箱和管理员预设的离线密码登录。
- 离线密码只保存哈希，不保存明文密码。

默认已经授权管理员本人：

```text
已在本机配置
```

新增员工时，管理员需要填写：

- 使用者姓名
- 使用者邮箱
- 离线密码
- 所属分行

## 在线和离线策略

建议正式使用时采用这个模式：

- 正常情况：联网使用，读取服务器最新版系统和云端数据
- 断网情况：继续用本机缓存收银，订单先保存在本机
- 恢复网络：把离线订单同步回云端总后台

当前版本已经完成第一步：系统文件“联网优先、断网可打开”。下一阶段要做的是“订单和库存云端同步”，这样总管理员才能看到所有分行的实时数据。

当前 Firebase 已经接入基础层：Google 登录、云端状态、销售记录写入 Firestore、分行和授权用户写入 Firestore、离线订单队列、收款后库存同步。在线收款会优先使用 Firestore transaction 同时写订单和扣库存，降低多设备同时销售时的库存冲突。

如果在线 transaction 因云端库存不足或权限问题失败，本机订单和库存会自动回滚，不会显示成功小票。

管理员后台已经加入 `初始化云端数据`。第一次完成 Firebase 设置后，管理员 Google 登录，再点击这个按钮，就会把本地分行、授权用户、商品上传到 Firestore。

系统也加入了 `刷新云端资料`，可手动从 Firestore 读取分行、授权用户、商品、销售记录、库存流水和审计日志，并保存成本机缓存。

后台现在也会显示：

- 同步状态：还有多少离线订单待补传
- 热销商品：按当前日期范围统计商品销量和销售额
- 每日销售趋势：按当前日期范围查看每日订单数和销售额
- 客户跟进：即将到期或刚到期的计划客户
- 低库存提醒：各分行库存小于或等于 5 的商品
- 业务设置：品牌名、小票标题、默认计划、服务天数、低库存提醒阈值、小票结尾
- 完整备份：导出分行、授权用户、商品、销售、待同步订单和业务设置
- 离线补传：订单和库存变更都会进入待同步队列
- 库存调整流水：记录商品、分行、调整前后库存、调整人和原因
- 审计日志：记录新增分行、授权用户、移除用户、业务设置、备份恢复、清空数据、云端初始化
- 审计和库存报表：管理员可导出 CSV，方便对账、追踪人为调整和留档
- 商品和库存报表：管理员可导出商品销售排行和各分行库存总表
- 客户跟进：管理员可标记已联系或已完成，销售 CSV 会包含跟进状态
- 离线跟进：客户跟进状态使用独立订单更新队列，恢复网络后只更新订单资料，不重复扣库存
- 备份恢复：先恢复到本机，确认正确后再手动初始化云端数据

离线订单队列工作方式：

- 断网或云端不可用时完成收款，订单会先保存在本机待同步队列。
- 顶部云端状态会显示待同步数量。
- 恢复网络并登录云端后，系统会自动补传待同步订单。

Firebase 配置安全：

- `firebase-config.example.js` 是模板，可以提交到 GitHub。
- `firebase-config.local.js` 放真实 Firebase 配置，只留本机，不提交。
- 前台页面不会公开显示完整管理员邮箱。
- `app.js` 只保存管理员邮箱哈希，不保存明文邮箱。
- 如果 GitHub 提醒 Google API Key 泄露，请到 Google Cloud 限制 API Key 的可用域名。

## 本地运行

如果电脑装了 Python，在这个文件夹运行：

```powershell
python -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

也可以直接双击 `index.html` 预览，但离线安装功能需要通过 `http://localhost` 或 HTTPS 网站访问。

## 上线方式

最简单的方式是把这些文件上传到任意静态网站服务，例如 Netlify、Vercel、Cloudflare Pages、GitHub Pages，或你的服务器静态目录。

上线时至少需要上传：

- `index.html`
- `styles.css`
- `app.js`
- `sw.js`
- `manifest.webmanifest`
- `icon.svg`

注意：正式上线必须使用 HTTPS，浏览器才会启用 PWA 缓存。当前缓存策略是联网优先，断网时才使用离线缓存。

## 数据保存在哪里

当前版本的数据保存在使用者这台设备的浏览器 `localStorage` 中，所以它天生支持离线，但不同电脑之间不会自动同步。

后续如果要做多门店、多设备同步，可以继续增加：

- 登录账号
- 云端数据库
- 库存同步
- 订单同步
- 权限管理
- 会员系统
- 扫码枪和小票打印机适配
