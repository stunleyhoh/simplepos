# 简单POS逐步测试记录

测试建议顺序：先测本机基础功能，再测 Firebase 云端，最后测离线同步。每完成一项就在前面打勾。

## 0. 测试前准备

- [ ] 使用 Chrome 或 Edge 打开系统。
- [ ] 打开页面后，顶部版本显示 `v0.62`。
- [ ] 如果页面仍显示旧版，按 `Ctrl + F5` 强制刷新一次。
- [ ] 确认 Firebase Google 登录已加入授权域名，例如 GitHub Pages 域名和 localhost。
- [ ] 确认 Firestore 规则已经发布。

## 1. 首页和菜单

- [ ] 打开后默认进入 `下单` POS 画面。
- [ ] 左下角菜单按钮可以打开。
- [ ] 左下角菜单顺序为：下单、菜单、库存、转账记录、报告、设置。
- [ ] 点击每个菜单后，页面内容会正确切换。
- [ ] 右下角收银菜单可以打开。
- [ ] 右下角显示已登录操作员、班次状态、员工登录 / 登出。

## 2. 员工登录和班次

- [ ] 联网时点击 `联网 Google 登录`，使用已授权 Google 邮箱登录 POS。
- [ ] 联网时直接提交邮箱密码表单，会提示必须使用 Google 登录。
- [ ] 管理员在 `设置` 里授权 / 更新员工时，需要设置至少 6 位离线密码。
- [ ] 断网时使用授权邮箱 + 离线密码登录 POS。
- [ ] 断网时输入错误离线密码，会拒绝登录。
- [ ] 登录后右下角显示当前操作员。
- [ ] 普通收银员登录后，系统自动切换到该员工授权分行。
- [ ] 普通收银员登录后，分行下拉被锁定，不能切换其他分行。
- [ ] 普通收银员可进入 `库存`，但只能调整自己授权分行库存。
- [ ] 普通收银员可进入 `转账记录`，但只能看到和退款 / 作废自己授权分行订单。
- [ ] 登录后自动开始班次，能看到班次开始时间。
- [ ] 结束班次 / 交班时会出现结算资料。
- [ ] 到 `设置` 页面可以看到交班记录。

## 3. 标准收银

- [ ] 选择当前分行。
- [ ] 点击 `简单草本减脂计划第一阶段` 加入购物车。
- [ ] 商品价格显示 RM 150。
- [ ] 实收金额自动预填为应收金额。
- [ ] 修改数量后，小计、应收、实收建议金额正确变化。
- [ ] 输入折扣后，应收金额和找零正确变化。
- [ ] 填写客户姓名和电话。
- [ ] 可选填联盟推荐码，完成订单后销售记录及 CSV 均保留推荐码。
- [ ] 选择付款方式 `现金`，完成收款。
- [ ] 出现小票，小票包含客户资料、付款方式、3 星期计划日期。
- [ ] 联网但网速慢时，完成收款不应长时间卡住，应先出现小票。
- [ ] 云端尚未完成时，销售记录同步状态可显示后台同步中或待同步。
- [ ] 今日销售出现刚刚完成的订单。

## 4. SimplePay 和非现金付款

- [ ] 新建一笔订单。
- [ ] 付款方式选择 `简单支付 / SimplePay`。
- [ ] 填写付款参考号。
- [ ] 完成收款后，小票显示 `简单支付 / SimplePay`。
- [ ] 到 `转账记录` 页面，订单显示付款方式和参考号。
- [ ] 到 `报告` 页面，付款方式汇总包含 SimplePay。
- [ ] 再测试一笔非现金付款但不填写参考号，系统会提醒确认。

## 5. 转账记录和作废

- [ ] 进入 `转账记录` 页面。
- [ ] 可以按日期筛选。
- [ ] 可以按付款方式筛选。
- [ ] 当天结算汇总会跟随筛选变化。
- [ ] 可以用客户、电话、订单号搜索。
- [ ] 订单超过 8 笔时，`显示更多` 能展开。
- [ ] 管理员作废订单时，需要输入 `VOID`。
- [ ] 作废后订单显示已作废。
- [ ] 作废订单不会进入销售报表统计。
- [ ] 作废后库存会回补。

## 6. 菜单、库存和分行

- [ ] 进入 `菜单` 页面，能看到现有商品。
- [ ] 搜索商品名、SKU 或分类，可以过滤商品。
- [ ] 点击商品后，编辑表单会带入该商品资料。
- [ ] 进入 `库存` 页面，可以先选择要查看的分行。
- [ ] 搜索商品名、SKU 或分类，可以过滤库存。
- [ ] 点击商品右侧库存按钮并调整数量后，库存流水新增记录。
- [ ] 新增测试分行。
- [ ] 授权一个测试 POS 用户到测试分行。
- [ ] 在库存页切换测试分行后，库存独立显示和调整。

## 7. 后台报告

- [ ] 管理员进入后台。
- [ ] 管理员登录后，分行下拉可切换总店和所有分行。
- [ ] 管理员切换任何分行后，都能查看该分行库存、菜单和订单资料。
- [ ] 管理员切换分行后，可以继续执行后台管理操作。
- [ ] 日期范围切换 `今天`、`本月`、`全部`，指标会变化。
- [ ] 分行表现会跟随日期范围变化。
- [ ] 热销商品会跟随日期范围变化。
- [ ] 每日销售趋势会跟随日期范围变化。
- [ ] 付款方式汇总会跟随日期范围变化。
- [ ] 客户跟进可以标记 `已联系`。
- [ ] 客户跟进可以标记 `已完成`。
- [ ] 已完成客户不再显示在待跟进列表。

## 11. 蓝牙打印机

- [ ] Chrome / Edge 可打开蓝牙配对清单，取消配对不会影响订单。
- [ ] 58mm 测试小票的中文、英文、数字均清晰且不乱码。
- [ ] 80mm 纸宽切换后测试小票宽度正确。
- [ ] 收款完成页可手动执行“蓝牙打印”，系统打印仍可使用。
- [ ] 开启自动打印后，收款成功只自动打印一次。
- [ ] 打印机关闭或断线时订单仍成功保存，并显示可理解的错误。
- [ ] 重新开启打印机后可用“连接已配对打印机”恢复连接。
- [ ] 移除打印机后，本机不再保留该设备资料。

## 12. 三系统低成本关联

- [ ] 新订单号包含 POS、分行标识、时间和随机尾码。
- [ ] 关联资料弹窗可以一键复制 POS 订单号。
- [ ] 旧订单打开后仍可正常显示，并自动补齐外部关联结构。
- [ ] SimplePay 订单可以补录参考号并标记“已关联”。
- [ ] “已关联”但没有 SimplePay 参考号时禁止保存。
- [ ] 联盟推荐码会转换为大写并保存在订单、收据和 CSV。
- [ ] 联盟订单可以补录订单号并标记“已关联”。
- [ ] “已关联”但没有联盟订单号时禁止保存。
- [ ] 断网时修改关联资料会显示“更新待同步”。
- [ ] 有待同步更新时刷新云端资料，本机更新不会被旧云端版本覆盖。
- [ ] 设置页融合准备状态会统计待确认、待关联和异常数量。
- [ ] 重复使用相同 SimplePay 参考号或联盟订单号时，设置页会显示关联异常。
- [ ] 转账记录可以筛选全部待处理、SimplePay 待确认、联盟待关联、已关联及异常。
- [ ] 待关联清单 CSV 包含所有日期，但普通员工只会导出授权分行订单。
- [ ] 重复补传同一订单不会再次扣减云端库存。
- [ ] 联网订单首次云端同步未完成时，系统会暂时阻止立即退款。
- [ ] 断网订单仍可退款 / 作废，并正确回补本机库存。
- [ ] 发布最新 Firestore 规则后，员工不能修改其他分行的 `branchStock`。
- [ ] 员工订单更新不能修改金额、商品、收银员或所属分行。
