# 简单 POS 系统逐步测试记录

测试建议顺序：先测本机基础功能，再测 Firebase 云端，最后测离线同步。每完成一项就在前面打勾。

## 0. 测试前准备

- [ ] 使用 Chrome 或 Edge 打开系统。
- [ ] 打开页面后，顶部版本显示 `v0.42`。
- [ ] 如果页面仍显示旧版，按 `Ctrl + F5` 强制刷新一次。
- [ ] 确认 Firebase Google 登录已加入授权域名，例如 GitHub Pages 域名和 localhost。
- [ ] 确认 Firestore 规则已经发布。

## 1. 首页和菜单

- [ ] 打开后默认进入 `下单` POS 画面。
- [ ] 左下角菜单按钮可以打开。
- [ ] 左下角菜单顺序为：下单、菜单、库存、转账记录、报告、设置。
- [ ] 点击每个菜单后，页面内容会正确切换。
- [ ] 右下角收银菜单可以打开。
- [ ] 右下角显示已登录操作员、班次状态、员工登录 / 登出。

## 2. 员工登录和班次

- [ ] 使用已授权邮箱登录 POS。
- [ ] 登录后右下角显示当前操作员。
- [ ] 登录后自动开始班次，能看到班次开始时间。
- [ ] 结束班次 / 交班时会出现结算资料。
- [ ] 到 `设置` 页面可以看到交班记录。

## 3. 标准收银

- [ ] 选择当前分行。
- [ ] 点击 `简单草本减脂计划第一阶段` 加入购物车。
- [ ] 商品价格显示 RM 150。
- [ ] 实收金额自动预填为应收金额。
- [ ] 修改数量后，小计、应收、实收建议金额正确变化。
- [ ] 输入折扣后，应收金额和找零正确变化。
- [ ] 填写客户姓名和电话。
- [ ] 选择付款方式 `现金`，完成收款。
- [ ] 出现小票，小票包含客户资料、付款方式、3 星期计划日期。
- [ ] 今日销售出现刚刚完成的订单。

## 4. SimplePay 和非现金付款

- [ ] 新建一笔订单。
- [ ] 付款方式选择 `简单支付 / SimplePay`。
- [ ] 填写付款参考号。
- [ ] 完成收款后，小票显示 `简单支付 / SimplePay`。
- [ ] 到 `转账记录` 页面，订单显示付款方式和参考号。
- [ ] 到 `报告` 页面，付款方式汇总包含 SimplePay。
- [ ] 再测试一笔非现金付款但不填写参考号，系统会提醒确认。

## 5. 转账记录和作废

- [ ] 进入 `转账记录` 页面。
- [ ] 可以按日期筛选。
- [ ] 可以按付款方式筛选。
- [ ] 当天结算汇总会跟随筛选变化。
- [ ] 可以用客户、电话、订单号搜索。
- [ ] 订单超过 8 笔时，`显示更多` 能展开。
- [ ] 管理员作废订单时，需要输入 `VOID`。
- [ ] 作废后订单显示已作废。
- [ ] 作废订单不会进入销售报表统计。
- [ ] 作废后库存会回补。

## 6. 菜单、库存和分行

- [ ] 进入 `菜单` 页面，能看到现有商品。
- [ ] 搜索商品名、SKU 或分类，可以过滤商品。
- [ ] 点击商品后，编辑表单会带入该商品资料。
- [ ] 进入 `库存` 页面，能看到各分行库存。
- [ ] 搜索商品名、SKU 或分类，可以过滤库存。
- [ ] 调整库存后，库存流水新增记录。
- [ ] 新增测试分行。
- [ ] 授权一个测试 POS 用户到测试分行。
- [ ] 切换到测试分行后，库存独立显示。

## 7. 后台报告

- [ ] 管理员进入后台。
- [ ] 日期范围切换 `今天`、`本月`、`全部`，指标会变化。
- [ ] 分行表现会跟随日期范围变化。
- [ ] 热销商品会跟随日期范围变化。
- [ ] 每日销售趋势会跟随日期范围变化。
- [ ] 付款方式汇总会跟随日期范围变化。
- [ ] 客户跟进可以标记 `已联系`。
- [ ] 客户跟进可以标记 `已完成`。
- [ ] 已完成客户不再显示在待跟进列表。

## 9. 离线同步

- [ ] 登录后断开网络。
- [ ] 页面刷新后仍能打开。
- [ ] 离线完成一笔收款。
- [ ] 顶部显示有待同步订单。
- [ ] 离线修改客户跟进状态。
- [ ] 顶部显示有订单更新待同步。
- [ ] 恢复网络。
- [ ] 点击补传离线订单，待同步数量下降。
- [ ] 刷新云端资料后，订单仍存在。

## 10. 数据保护

- [ ] 完整 JSON 备份可以导出。

- [ ] 清空全部数据时，必须输入 `RESET`。
- [ ] 页面不会公开显示完整管理员邮箱。
