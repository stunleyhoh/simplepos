# 简单POS三系统低成本融合方案

## 当前阶段

三个 Firebase 项目继续独立：

- 简单POS：订单、分行、库存、收银和收据。
- 1分钟支付：钱包、商家收款、退款和结算。
- 一分钟联盟营销系统：推荐关系、配套、奖励和提现。

当前只在 POS 订单保存关联资料，不持续监听或读取另外两个项目。

## 统一订单关联

每张新 POS 订单包含：

- `externalReferences.posOrderId`
- `externalReferences.simplePayReference`
- `externalReferences.simplePayStatus`
- `externalReferences.affiliateReferralCode`
- `externalReferences.affiliateOrderId`
- `externalReferences.affiliateStatus`

旧订单会在本机自动补齐这些字段，不会改变原订单金额、库存或付款状态。

## 状态定义

- `not-used`：该订单没有使用对应系统。
- `pending`：已经取得推荐码或选择 SimplePay，但尚未完成外部确认。
- `linked`：已经人工核对并填写外部订单或付款参考号。
- `failed`：资料不一致，需要人工复核。

## 当前人工操作流程

1. 在简单POS下单；客户有联盟推荐码时，在“订单设置”填写。
2. 付款方式选择 SimplePay 时，先完成顾客付款。
3. 已取得 SimplePay 交易号就在收款参考号填写；尚未取得可以先完成 POS 订单，系统会标记“待确认”。
4. 到“转账记录”找到订单，点击“关联资料”。
5. 可以复制 POS 订单号，粘贴到 SimplePay 或联盟订单备注，减少人工抄写错误。
6. 核对 SimplePay 后填写参考号并改为“已关联”。
7. 联盟订单建立后填写联盟订单号并改为“已关联”。
8. 到“设置 > 三系统融合准备”查看剩余待确认、待关联和异常数量。
9. 可在“转账记录”按关联状态筛选，或导出全部日期的待关联清单集中处理。

## 成本控制

- POS 收银不等待 SimplePay 或联盟系统。
- 不建立跨项目全局实时监听。
- 只有实际订单发生或人工更新关联资料时，才写入 POS 项目。
- SimplePay 和联盟项目暂不因 POS 页面刷新而产生读取。
- 外部系统故障时，POS 仍可现金收款、离线保存和打印小票。

## POS 权限保护

- 员工只能更新自己授权分行的销售订单。
- 员工不能通过订单更新修改金额、商品、收银员或分行。
- 员工库存更新只能改变 `branchStock` 中自己授权分行的键。
- 总店员工更新总店库存时，才允许同步旧版 `stock` 字段。
- 管理员仍保留全局维护权限。

## 上线前阻挡项

1. SimplePay 余额变化仍由浏览器执行，Firestore 规则不适合正式资金业务。
2. 联盟 Cloud Function 当前为测试即时模式。
3. POS 示例商品 RM 150 与联盟配套 RM 180 不同；正式同步价格时以联盟配套为准。
4. 自动确认付款及自动发放奖励必须放在可信后台，不能由浏览器直接写余额。

## 后续阶段

1. SimplePay 增加不可重复的外部订单号。
2. SimplePay 安全后台确认付款后，按 POS 订单号返回结果。
3. 联盟系统接收已确认付款订单并产生奖励。
4. 最后才加入统一登录和自动对账。
